LWP3-GT4-SDK v2.0 "The ADAS Refactor"
====================================

NEW FEATURES:
- ASYNCHRONOUS ARCHITECTURE: Logic and Bluetooth comms are now decoupled into 
  dedicated threads, achieving a jitter-free 50Hz control loop.
- MAILBOX PATTERN: Implemented a sequence-based TX mailbox. The hardware 
  thread consumes the newest control batch every 2ms, preventing packet 
  congestion while ensuring the lowest possible command latency.
- MECHANICAL-AWARE CALIBRATION: Uses a 5-step sliding window algorithm to find 
  physical chassis limits, intelligently handling gear-clutch slippage and 
  static-friction wedging.
- ADAS WATCHDOG: Monitors TX success rates to detect link degradation. If 
  communication is lost, the car executes an emergency stop within 200ms.
- VIRTUAL PORT OPTIMIZATION: Automatically combines Left and Right motor 
  channels when driving straight, halving the Bluetooth traffic load.

FIXES:
- Resolved the "Const-Correctness" compilation error on modern GCC.
- Eliminated core dumps on shutdown by ensuring object initialization checks.
- Fixed the "Virtual Port Twitch" surge during SDK disconnect.

LWP3-GT4-SDK v2.1 "Implement delta-encoding"
====================================

- Added send-on-change delta encoding to the 50Hz control loop, reducing BLE traffic by ~90% and completely eliminating BlueZ D-Bus timeouts.
- Verified physical steering response latency at ~258ms.
- Added graceful Bluetooth unsubscription on disconnect to prevent D-Bus warnings.
- Finalized the 5-step mechanical-aware calibration sequence with static friction un-wedging.

Version: 3.0.0 "Async Pipeline"
====================================

SUMMARY:
Version 3.0.0 represents a complete architectural overhaul, transitioning the 
SDK from a synchronous callback-driven library to a high-performance, 
asynchronous Real-Time Hardware Abstraction Layer (HAL).

NEW FEATURES:
- Asynchronous API: Commands and Telemetry are now decoupled via thread-safe 
  queues (TSQueue). Application logic can now run at deterministic frequencies 
  (e.g., 50Hz) regardless of BLE jitter.
- Triple-Threaded Architecture:
    1. BLE RX Thread (SimpleBLE): Ingests raw hardware packets.
    2. Logic/Damping Thread (50Hz): Handles rate-limiting and gear protection.
    3. BLE TX Thread (66Hz): Manages high-speed D-Bus command dispatch.
- Soft-Margin Protection: Calibration now implements a 'SOFT_MARGIN' that 
  prevents the steering servo from physically impacting plastic end-stops.
- Safety Watchdog: Automatic throttle cut-off if the 'Brain Pulse' (Autonomy 
  command) is not received within 200ms.
- Enhanced Telemetry: Telemetry packets now include Battery Level, Link Health 
  status, and high-resolution Monotonic Timestamps for Kalman Filter integration.

REMOVED/DEPRECATED:
- Deprecated 'onSteerUpdate' and 'onBatteryUpdate' callbacks.
- Removed 'updateControl' in favor of 'setCommand'.
- Removed blocking 'isLinkHealthy' calls in favor of Telemetry packet polling.

FIXES:
- Eliminated "Start-Stop" jerky motion by enforcing a steady 50Hz heartbeat.
- Resolved BlueZ D-Bus timeouts during high-frequency command bursts.

Version: 4.9.0 "The Deterministic HAL"
====================================

SUMMARY:
Version 4.9 completes the transition of the SDK into a production-grade 
Hardware Abstraction Layer (HAL). It removes all "smart" internal control 
logic (smoothing, PID-like behavior) to provide the raw determinism required 
for high-speed autonomous state estimation and path tracking.

CORE ARCHITECTURE:
- Deterministic Timing: Every Telemetry packet and TxEvent now includes 
  nanosecond-precision CLOCK_MONOTONIC_RAW timestamps.
- Zero Control Policy: The HAL no longer alters user commands for "feel". 
  Trajectory smoothing is now the responsibility of the ADAS layer.
- Thread-Safe Pipeline: Non-blocking command enqueuing and lock-free 
  telemetry access.

MECHANICAL SAFETY:
- V3-Robust Calibration: Restored the 5-step gentle calibration sequence 
  that resets tension between limit tests.
- Hardware Safety Ramp: An internal protection fuse limits steering deltas 
  to 15 units per 15ms to protect plastic gears from accidental torque snaps.
- Soft-Margin Enforcement: Detected limits are padded by 4 units to prevent 
  physical chassis impacts during autonomous operation.

STABILITY FIXES:
- BlueZ Hardening: Added GATT service discovery loops to prevent 
  SimpleBLE "Not Initialized" crashes on Linux.
- Motor Protocol: Standardized on WriteDirectModeData (0x51) for high-speed 
  drive and StartSpeed (0x01) for calibration.
- Persistent Connection: Discovery loop now waits for Hub button press.

Version: 5.0.0 "The Hardened HAL"
====================================

This release fulfills the 'Tightening' pass for ADAS-ready operation.

KEY IMPROVEMENTS:
- Lock-Free Core: Telemetry and Command access now use atomic latches, 
  eliminating mutex contention in the high-frequency control path.
- Non-Blocking TX: Replaced sleep-based rate limiting with a 
  high-frequency time-gate (Busy-wait/Yield). This ensures deterministic 
  command dispatch precisely as the hardware window opens.
- Robust Diagnostics: Latency profiling now uses Epsilon-matching to 
  account for mechanical slack and includes automatic stale-packet cleanup.
- ADAS Lead: Zero hidden control policy. Steering safety ramps are 
  disabled by default.
- BlueZ Hardening: Optimized disconnection sequence and detached callback 
  threads to accommodate Experimental BlueZ backend jitter.

Version: 7.0.0 "The Mechanical Truth"
====================================

This release represents the shift from a communication-focused driver to a
Control-System-Aware HAL.

Key Improvements
- True Latency Profiling: Introduced Epsilon Matching. We no longer report
  the speed of the Bluetooth air (10ms). We now report the speed of the
  physical steering rack (90ms mean), enabling the next stage of ADAS:
  Latency Compensation.
- Deterministic Control: Replaced thread-sleeping with a Timestamp Rate Gate.
  The TX thread remains "hot" and responsive, firing the moment the 15ms
  BlueZ stability window opens.
- Mechanical Integrity: Set ϵ=3.0 as the production threshold. This value was
  found through sensitivity analysis to be the "Mechanical Plateau" of the
  Porsche GT4's steering rack, effectively filtering out gear-lash and jitter.
- Memory Efficiency: Cleaned public surface area. The HAL is now an opaque,
  deterministic pipe.
